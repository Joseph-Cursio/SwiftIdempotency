# SwiftIdempotency 0.3.1

Dependency-friction fix. Same-day follow-up to v0.3.0 — no API changes,
no behavior changes.

## Fixed

### `swift-syntax` dependency constraint loosened from `exact: "602.0.0"` to `"602.0.0"..<"604.0.0"`

The `exact: "602.0.0"` pin blocked adopters whose other dependencies
transitively required swift-syntax 603 (notably DiscordBM's
`@UnstableEnum` macro, which only expands correctly under swift-syntax
603+). SwiftPM's resolver would honor the `exact` constraint and
downgrade the whole graph to 602, breaking the 603-only macro
expansion at compile time.

The range form `"602.0.0"..<"604.0.0"` lets SwiftPM pick whichever
version the adopter's full dependency graph agrees on, within a band
the SwiftIdempotency macros are verified against. Both 602.0.0 and
603.0.0 were test-suite-validated before release (74/74 green under
each). 604 is deliberately excluded pending verification on the new
major — a future patch release will extend the upper bound once
that's done.

### Surfacing adopter

Penny's `package-integration-trial` branch was the surfacing signal:
the bug-sweep trial's pin bump from the v0.3.0-prototype SHA to the
`0.3.0` tag triggered a clean re-resolve, which propagated the
`exact` constraint and broke DiscordBM. The prior coin-double-grant
trial had passed because its Package.resolved predated the pin change
and still listed swift-syntax 603.

## Validation

- **Root test suite:** 74/74 green under both swift-syntax 602.0.0 and
  603.0.0. No API surface change; no test additions/removals.
- **Adopter validation:** resumed the Penny bug-sweep trial after the
  0.3.1 pin bump resolved the DiscordBM compile failure. See
  [`docs/penny-package-trial/`](../penny-package-trial/) for the
  bug-sweep artifacts.

## Migration from v0.3.0

**No action required.** Adopters on v0.3.0 can upgrade freely; the
constraint change is strictly additive (more versions allowed, no
versions removed). Adopters currently blocked by the `exact: "602"`
pin get unblocked by moving to `from: "0.3.1"`.
